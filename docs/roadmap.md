# Roadmap

## Near Term

- tighten normative language in the specification
- resolve any remaining root-object encoding ambiguities
- add dedicated conformance tests
- publish cleaner examples for maps, arrays, and schema overrides
- add repository automation for build verification

## Mid Term

- package the .NET library for easier consumption
- publish a supported TypeScript implementation
- add round-trip tests across implementations
- define compatibility and deprecation policy

## Longer Term

- additional language implementations
- compatibility test corpus
- formal release process for protocol revisions
